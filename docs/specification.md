# 美洲地形與生態互動式教材規格書

## 一、專案目標

建立一套可部署在 GitHub Pages 的美洲互動式地理教材。學生以互動地圖探索山脈、水文、特殊地形、國家公園與代表性動植物，透過圖層切換理解地理環境與生態分布的空間關係。每個項目、外部實景與照片都必須附上來源。

## 二、適用對象與學習目標

適用於國中地理課程，亦可延伸至高中。學生完成教材後應能：

1. 辨識美洲主要山脈、高原、平原與重要生態區。
2. 說明主要河川、湖泊、流域與分水嶺。
3. 分辨河蝕、冰河、火山、風成及喀斯特地形。
4. 解釋地形與氣候、水文、生態及人類活動的關係。
5. 認識美國、加拿大、墨西哥及中美洲代表性國家公園。
6. 正確引用官方資料、開放資料與影音來源。

## 三、核心系統功能

### 3.1 地圖

- 使用 Leaflet 建立互動地圖。
- 支援縮放、拖曳、景點點擊與標籤提示。
- 預設範圍涵蓋加拿大、美國、墨西哥、格陵蘭與中美洲。

### 3.2 底圖切換

第一版提供：

- OpenStreetMap 一般地圖。
- OpenTopoMap 地形圖。
- Esri World Imagery 衛星影像。

所有底圖必須保留原始 attribution。後續可再加入符合授權的地形陰影、土地覆蓋或歷史地圖。

### 3.3 資料圖層

學生可開關：

- 山脈景點。
- 河流景點。
- 湖泊景點。
- 特殊地形景點。
- 國家公園景點。
- 山脈範圍 GeoJSON。
- 主要河流 GeoJSON。
- 主要湖區 GeoJSON。

第二階段可加入：地形區、流域、板塊邊界、火山、冰河、沙漠、永久凍土與國家公園界線。

### 3.4 景點資訊卡

每筆至少包含：

- 中文名稱與英文名稱。
- 類別、國家與地形區。
- 經緯度。
- 地形簡介。
- 形成原因。
- 觀察重點。
- 思考問題。
- Google Maps 或其他實景連結。
- YouTube 公開影片或搜尋連結。
- 官方／主要資料來源。
- 授權或來源說明。

### 3.5 搜尋與導覽

- 可用中英文名稱搜尋景點。
- 搜尋結果可將地圖移至景點並打開資訊卡。
- 自動導覽依序介紹代表性山脈、河流、湖泊、特殊地形與公園。

### 3.6 學習活動

- 地圖觀察任務。
- 隨機選擇題。
- 未來可加入拖曳配對、河流追蹤、分水嶺判斷與國家公園護照。

## 五、資料架構

景點存放於 `data/places.json`；測驗存放於 `data/quizzes.json`；線面圖層存放於 `geojson/`。資料與畫面程式分離，以便教師或開發者增修內容。

## 六、公開資料與來源策略

1. Natural Earth 作為公開領域地理向量資料的重要來源。
2. OpenStreetMap、OpenTopoMap 與 Esri 作為底圖服務，依各自條款顯示 attribution。
3. NPS、Parks Canada、USGS、NOAA、Natural Resources Canada 與 INEGI 作為內容查核與延伸閱讀來源。
4. YouTube 僅嵌入或外連公開內容，不下載或重新散布。
5. Google Maps 第一版使用搜尋外連，避免要求教師設定 API Key。
6. 教學簡化圖層需清楚標示不是測量級資料。

## 七、介面需求

### 桌面版

左側為分類、搜尋、任務與圖例；中央為地圖；右側為景點資訊卡。

### 行動版

分類控制顯示於地圖上方，地圖保持主要畫面，資訊卡移至地圖下方。

### 無障礙

- 按鈕具可辨識文字或 `aria-label`。
- 文字與背景保持足夠對比。
- 不只使用顏色區分類別，另搭配圖示與文字。
- 外部連結標示開新視窗。

## 八、非功能需求

- 純前端，不需要後端或資料庫。
- 可部署於 GitHub Pages。
- 不收集姓名、位置或作答個資。
- 首頁與核心資料在一般校園網路下可快速載入。
- 外部服務失效時，核心文字教材仍可閱讀。
